# 午间检查

使用直接、清醒的语气。如果上午已经明显跑偏，不要粉饰；要把真正的问题指出来。

以下路径都相对当前 OpenClaw workspace 根目录。

## 记录分流规则

- 今天上午实际发生的事实、进展、阻塞 → 写进今日日记
- 下午或以后要处理的动作 → 放进 `50-59-capture/51-inbox/inbox.md`
- 已形成的明确结论 → 视类型沉淀到决策 / 研究 / 人物笔记

不要把整段聊天原样存档；先提炼，再落到正确位置。

## 第 1 步：加载上下文

并行阅读以下内容：

1. 今天的日记：`20-29-journal/21-daily/YYYY/MM/YYYY-MM-DD.md`。
2. 年度目标：`10-19-planning/12-annual-goals/YYYY.md`。
3. Inbox：`50-59-capture/51-inbox/inbox.md`。
4. 如有需要，再读人生规划：`10-19-planning/11-life-plan/plan.md`。
5. `30-39-knowledge/32-decisions/` 中状态为 `Open` 或 `Active` 的决策。

## 第 2 步：检查上午的真实情况

先呈现：早上原本最重要的事情是什么、现在实际推进到了哪里、哪些地方卡住了。

不要只接受泛泛而谈。优先使用宿主提供的结构化提问 / 澄清工具；如果没有，就直接追问：

- 哪件事已经完成？
- 哪件事其实没动？
- 当前阻塞是客观问题，还是在回避困难？

## 第 3 步：更新今天的日记

把上午值得保留的事实写回今天的日记：

- 在“记录”里补必要的时间戳日志
- 填写或刷新“午间检查”部分
- 如果早上的承诺已经失真，就明确改掉，而不是假装它还成立

## 第 4 步：重排下午

帮助用户把下午重新收敛到最关键的 1 到 2 件事：

1. 哪件事今天必须继续推进？
2. 哪件事应该降级、推迟，或放进 inbox？
3. 哪个阻塞需要今天主动求助、决策或拆小？

如果清单继续膨胀，就直接指出来。

## 第 5 步：收尾

- 保留一个现实的下午承诺，不要重新立一份漂亮但做不完的计划
- 把不属于今天的动作放进 `50-59-capture/51-inbox/inbox.md`
- 最后用一句话总结：今天下午真正要守住的是什么
