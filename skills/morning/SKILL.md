# 晨间记录、复盘与规划

使用直接、好奇的语气。如果计划模糊、过载，或者和用户目标脱节，要明确指出来。

以下路径都相对当前 OpenClaw workspace 根目录。

## 记录分流规则

- 今天的事实、进展、阻塞 → 写进今日日记
- 后续动作、提醒、待办 → 放进 `50-59-capture/51-inbox/inbox.md`
- 已形成的明确结论 → 视类型沉淀到决策 / 研究 / 人物笔记

不要整段转存聊天原文；只提炼值得保留的内容。

## 第 1 步：加载上下文

并行阅读以下内容：

1. 昨天的日记：`20-29-journal/21-daily/YYYY/MM/YYYY-MM-DD.md`。如果不存在，就最多向前回看 7 天。
2. 今天的日记，路径规则相同。
3. 年度目标：`10-19-planning/12-annual-goals/YYYY.md`。
4. 人生规划：`10-19-planning/11-life-plan/plan.md`。
5. Inbox：`50-59-capture/51-inbox/inbox.md`。
6. `30-39-knowledge/32-decisions/` 中状态为 `Open` 或 `Active` 的决策。
7. 如果需要结构性上下文，就读 JD 索引：`00-09-system/00-index.md`。

## 第 2 步：回顾昨天

先呈现：昨天原本计划了什么、实际发生了什么、还有什么看起来没解决。

然后请用户带你走一遍昨天。优先使用宿主提供的结构化提问 / 澄清工具；如果没有，就直接明确追问。不要停留在模糊叙述上；一旦看到回避、过度承诺或自我糊弄，要直接说出来。

如有需要，更新昨天的日记：

- 补齐缺失的日志条目
- 如果晚间反思没写，就补上
- 标记已经完成的待办
- 保留用户原本的表达风格

## 第 3 步：检查今天

打开或创建今天的日记；如果文件不存在，就基于 `90-99-system/91-templates/daily-journal.md` 创建。

把它整理干净：

- 删除已经过期的延续事项
- 只保留今天仍然相关的承诺
- 把今天值得提升优先级的 inbox 项目提出来
- 如果本周相关，就刷新当前活跃决策

询问今天最重要的事情是什么。你可以给出选项，但不要让清单无限膨胀。

## 第 4 步：做对齐检查

检查今天的工作是否真正向上对齐：

1. 主要待办是否连回年度目标？
2. 有没有被长期忽略、最近完全没有动作的年度目标？
3. 当前活跃决策本周是否需要推进？
4. 如果优先级已经漂移，应该更新的是日程、目标，还是人生规划？

用 Johnny.Decimal 结构帮助用户重新定向。如果某样东西根本没有清晰归宿，要把这个问题指出来。

## 第 5 步：完成今天的日记

确保今天的笔记至少包含：

- 早晨的优先事项
- 日志区
- 当前项目
- 当前决策
- 只有在仍然相关时才保留延续事项

关键结果数量要现实一点。通常 3 个已经足够。

## 第 6 步：收尾

结束前：

- 质疑那些明显过于乐观的计划
- 点出任何正在被回避的重要事情
- 把不属于“今天”的后续动作放进 `50-59-capture/51-inbox/inbox.md`

最后用一句简短总结收尾：今天最重要的 1 到 3 件事是什么。

这份 skill 只负责晨间启动；午间检查与晚间总结分别交给 `midday` 和 `evening`。
