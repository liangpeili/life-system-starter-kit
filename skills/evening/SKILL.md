# 晚间总结

使用直接、平静的语气。不要把“今天很忙”误当成复盘；要把完成、偏差和回避说清楚。

以下路径都相对当前 OpenClaw workspace 根目录。

## 记录分流规则

- 今天已经发生的事实、结果、反思 → 写进今日日记
- 明天以后要跟进的动作 → 放进 `50-59-capture/51-inbox/inbox.md`
- 已形成、值得长期回看的结论 → 视类型沉淀到决策 / 研究 / 人物笔记

不要机械保存整段聊天记录；只保留经过提炼的事实、行动和结论。

## 第 1 步：加载上下文

并行阅读以下内容：

1. 今天的日记：`20-29-journal/21-daily/YYYY/MM/YYYY-MM-DD.md`。
2. 年度目标：`10-19-planning/12-annual-goals/YYYY.md`。
3. Inbox：`50-59-capture/51-inbox/inbox.md`。
4. 如有需要，再读人生规划：`10-19-planning/11-life-plan/plan.md`。
5. `30-39-knowledge/32-decisions/` 中状态为 `Open` 或 `Active` 的决策。

## 第 2 步：回读今天

先呈现：早上承诺了什么、午间后来怎么调整、现在实际完成了什么。

如果今天日志太稀薄，先帮助用户补上关键事实，再进入总结。

## 第 3 步：完成晚间部分

在今天的日记里完成“晚间”部分：

- 回答富兰克林的问题：**“今天我做了哪些有益的事？”**
- 写下简短反思：真正推进了什么、偏差在哪里、学到了什么
- 如果某件事没做完，先判断它是否仍值得延续

## 第 4 步：处理尾巴

不要机械把所有未完成事项抄到明天。先区分：

1. 明天仍然重要，值得延续
2. 只是噪音，应该丢掉
3. 不属于明天，应该放进 `50-59-capture/51-inbox/inbox.md`

如果某个年度目标持续没有动作，要明确指出来。

## 第 5 步：收尾

- 给出一句诚实的日终总结，不要写成空洞鸡汤
- 如果明天已有明显优先事项，可以留下 1 句提醒，但不要提前把明天排满
